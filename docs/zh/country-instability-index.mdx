国家不稳定指数（CII）为 Strategic Risk API 追踪的 31 个 Tier-1 国家维护一个高频不稳定性评分。CII 并不依赖静态评级，而是将编辑基线与实时事件压力相融合，并通过服务端
`GetRiskScores` RPC 发布结果。

完整的 v8 公式、系数表与变更历史，请参见
[CII Risk Scoring Methodology](/zh/methodology/cii-risk-scores)。本页面为面向运维人员的概览。

## 监测国家

当前 Tier-1 集合为：

| 区域 | 国家 |
|--------|-----------|
| **美洲** | 美国、委内瑞拉、巴西、墨西哥、古巴 |
| **欧洲** | 德国、法国、英国、波兰 |
| **东欧** | 俄罗斯、乌克兰 |
| **中东** | 伊朗、以色列、沙特阿拉伯、阿拉伯联合酋长国、土耳其、叙利亚、也门、伊拉克、黎巴嫩、埃及、卡塔尔 |
| **亚太** | 中国、台湾、朝鲜、印度、巴基斯坦、缅甸、韩国、日本 |
| **中亚/南亚** | 阿富汗 |

Tier-1 成员是人工遴选而非算法生成的。当一个国家具备持续的全球风险相关性、活跃/近期的武装冲突或严重国内动荡、较高的地区升级潜力、重大的经济/安全体系重要性，或仪表盘特定的监测需求时，即被纳入。对该集合的变更具有方法论层面的重大意义，因为 Strategic Risk 汇总分是根据该集合内评分最高的若干国家计算得出的。

## 评分模型

每个国家的评分将静态基线与动态事件评分相结合：

```
eventScore = Unrest * 0.25
           + Conflict * 0.30
           + Security * 0.20
           + Information * 0.25

combinedScore = baselineRisk * 0.40
               + eventScore * 0.60
               + supplemental boosts
```

结果会受到活跃冲突与旅行警示下限的钳制，随后约束至 0-100 区间。

| 组成 | 衡量内容 | 主要输入 |
|-----------|------------------|-------------|
| **Unrest** | 开放冲突之前或以下的民间动乱压力 | ACLED 抗议与骚乱、抗议致死人数、高严重度动乱、互联网/电力中断 |
| **Conflict** | 动能暴力与打击活动 | ACLED 战斗、爆炸、针对平民的暴力、致死人数、伊朗地区打击强度、以色列 OREF 警报 |
| **Security** | 该国附近的硬安全活动节奏 | 军用航班、军用舰船、航空关闭/延误、GPS/GNSS 干扰 |
| **Information** | 信息环境压力 | 机密新闻头条与按国家归因的威胁摘要 |

## 加成与下限

当基础事件组成之外出现佐证信号时，补充加成可抬升综合评分：

| 加成 | 上限 | 触发条件 |
|-------|----:|---------|
| 气候异常 | 15 | 按国家归因的气候严重度 |
| 网络威胁 | 12 | 按严重度加权的网络威胁计数 |
| 野火 | 8 | 高亮度与总火点计数 |
| 旅行警示 | 15 | 切勿旅行、重新考虑或谨慎类警示 |
| OREF 混合 | 25 | 活跃与 24 小时以色列警报压力 |
| 流离失所 | 20 | 对数缩放的人道主义流离失所 |
| 新闻紧迫度 | 5 | 高 Information 组成压力 |
| 地震 | 25 | 显著、重大或严重地震 |
| 制裁 | 14 | 制裁进入量与新进入活动 |
| AIS 中断 | 10 | 海事中断分桶 |

评分下限取 UCDP 冲突下限与国务院警示下限中的较大值。当实时警示数据缺失时，评分器会在计算警示加成与下限之前，应用
[方法论](/zh/methodology/cii-risk-scores#advisory-fallback-table) 中发布的精选兜底等级。

| 下限 | 阈值 |
|-------|----------:|
| UCDP 活跃战争 | 70 |
| UCDP 轻度冲突 | 50 |
| 切勿旅行警示 | 60 |
| 重新考虑旅行警示 | 50 |

UCDP 冲突下限源自**最新可用的** UCDP GED 版本（种子器选择返回事件的最新版本，而非仅仅第一个响应的版本），并且只统计**2 年时效窗口**内的事件——更早的事件不再锁定下限。在该窗口内，当一个 Tier-1 国家死亡总数大于 1000 或事件数大于 100 时，CII 将其归类为 UCDP **活跃战争**；当事件数大于 10 但未达到战争阈值时，归类为 UCDP **轻度冲突**。由于 UCDP GED 以年度版本发布、滞后于实时，近期的动能冲突由 **ACLED**（实时）与跨源信号源承载；UCDP 提供历史基线下限。在 `riskScores` 健康/缓存信号中，当 ACLED 或窗口内的 UCDP 源任一存在时，即视为冲突实时族已覆盖。ACLED 鉴权未配置会将冲突信号降级为仅 UCDP；只有当 ACLED 静默且 UCDP 也缺失、过期或超出 2 年评分窗口时，才变为 `COVERAGE_PARTIAL`（参见[数据源](/zh/data-sources#dual-source-protest-tracking)）。

## 不稳定性等级

| 等级 | 评分区间 | 含义 |
|-------|-------------|---------|
| **严重** | 81-100 | 活跃危机或重大升级 |
| **高** | 66-80 | 需密切监测的显著不稳定性 |
| **升高** | 51-65 | 高于常态的活动模式 |
| **正常** | 31-50 | 基线地缘政治活动 |
| **低** | 0-30 | 异常平静期 |

## 偏差预防

CII 有意不采用原始媒体体量指数。v8 模型将新闻压力与冲突评分分离，在高可观测性国家应用更低的事件乘数与对数衰减，并使用冲突/警示下限，以避免活跃危机在数据空窗期看似平静。

编辑依据、Tier-1 纳入标准、警示兜底表、加沙文本归因注意事项，以及按国家的 `baselineRisk` /
`eventMultiplier` 数值发布于
[CII Risk Scoring Methodology](/zh/methodology/cii-risk-scores)。

## 服务端预计算

CII 评分由 `GET /api/intelligence/v1/get-risk-scores` 在服务端计算，并缓存在 Redis 中。实时缓存按当前
`methodology_version`（`v8`）进行版本化，并在上游中断时使用一个 stale-cache 键作为兜底。

Railway 中继进程还每 8 分钟对同一 RPC 运行一次活跃 CII warm-ping 循环。这使服务端风险评分缓存在 bootstrap 与健康监测期间保持热状态，同时保留 RPC 处理器作为评分的权威来源。

### 支撑端点

| 相关输入 | 端点 | 备注 |
|---|---|---|
| CII / Strategic Risk 评分 | `GET /api/intelligence/v1/get-risk-scores` | 服务端评分权威来源，由中继循环保持热状态。 |
| 国家人道主义摘要 | `GET /api/conflict/v1/get-humanitarian-summary` | 用于冲突上下文工作流的 HAPI/HDX 国家概览。 |
| 批量人道主义摘要 | `POST /api/conflict/v1/get-humanitarian-summary-batch` | 多国家批量查询；使用 POST 因为请求携带国家列表。 |
| 人口暴露 | `GET /api/displacement/v1/get-population-exposure` | 人道主义与地图工作流使用的国家人口数据或基于半径的暴露估算。 |

## 趋势检测

服务器将 `dynamicScore` 发布为有符号变动 delta，范围
`-100..100`，基准为约 24 小时前的有效 CII 快照：

- **上升**：评分变动大于 1 个点。
- **稳定**：评分变动在 -1 到 +1 个点之间，或无有效的前序
  快照。
- **下降**：评分变动小于 -1 个点。

由于 CII `combinedScore` 取整到整数点，`+2` 与 `-2` 是
产生上升/下降趋势标签的首批整数变动。`dynamicScore`
在冷启动期间，或服务器无法找到约 24 小时前的有效前序快照时为 `0`。

在仪表盘初始启动期间，学习模式会在数据与趋势基线稳定之前抑制嘈杂的早期 CII 警报。服务端预计算意味着用户仍会立即收到评分；学习模式主要影响本地的告警生成。
