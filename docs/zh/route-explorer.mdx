Route Explorer 是一个全屏工作流，用于推理特定运输如何在世界中流动：它穿过哪些海上咽喉要道、存在哪些替代航线、哪些陆路走廊可以替代，以及一旦某个咽喉要道关闭，哪些国家会承受最大的冲击影响。这是回答"如果这条航线被阻断，实际会发生什么"的最快方式。

## 适用人群

- **货主、贸易商和运务台**——评估特定起运地-目的地-大宗商品组合的航线风险。
- **分析师**——在阻断情景下对比基线航线与合理的绕行方案。
- **政策与风险团队**——从国家层面审视进口依赖度与敞口。

## 打开 Route Explorer

- **Cmd / Ctrl + K → "Route Explorer — plan a shipment"**（`src/config/commands.ts:254`）。
- 或在命令面板中使用专属操作；该工作流会以全屏模态形式覆盖在地图之上打开。

路由状态编码在 URL 中（`src/components/RouteExplorer/url-state.ts`），因此任何配置——起运地、目的地、HS2 大宗商品、货物类型、当前活动标签页——都可以被复制、分享或加入书签。

## 四个标签页

Route Explorer 组织为四个标签页，定义于 `src/components/RouteExplorer/RouteExplorer.ts`：

### 1. 当前航线

所选起运地与目的地之间的基线航线，在地图上渲染并高亮显示咽喉要道穿越事件。展示该航线触及哪些咽喉要道、每个咽喉要道的实时阻断评分，以及附加在主要咽喉要道上的实时战争风险等级。

### 2. 替代方案

当主航线受损时的替代海上走廊——例如，绕行好望角以避开苏伊士 + 曼德海峡体系。每个替代方案都会显示增加的运输天数和附加成本乘数，并过滤出真正适用于所选货物类型的走廊。

### 3. 陆路

当海上航线存在可靠的铁路或公路对应方案时的陆路走廊替代（例如经中间走廊的中国–欧洲路线）。适用于思考陆海再平衡，而非单纯的海上绕行。

### 4. 影响

将问题从"我的货物去哪里"翻转为"如果这条航线中断，谁受冲击最大"。按各国通过所选咽喉要道对所选 HS2 章节的进口依赖度进行排名。

## 输入项

四个标签页共享同一输入栏：

| 输入项 | 取值 |
|---|---|
| **起运国** | 任意国家；底层使用 ISO-3166-1 alpha-2。 |
| **目的国** | 同上。 |
| **HS2 大宗商品** | 2 位 HS 章节（例如 `27` — 矿物燃料）。默认根据货物类型推断。 |
| **货物类型** | `container`、`tanker`、`bulk`、`roro`——将绕行走廊过滤为能承载该货物的走廊。 |

每次更改都会触发防抖重新拉取（约 250ms），因此按住按键或快速切换选项时感觉是实时的。

## 键盘优先

Route Explorer 为键盘使用而构建——所有输入和标签页都无需鼠标即可访问。完整的快捷键列表显示在工作流内的帮助浮层中（按 `?`），并位于 `src/components/RouteExplorer/KeyboardHelp.ts`：

| 按键 | 操作 |
|---|---|
| `Esc` | 先关闭当前活动的选择器，再关闭面板 |
| `Tab` / `Shift+Tab` | 在面板与地图之间移动焦点 |
| `F` | 跳转至起运国选择器 |
| `T` | 跳转至目的国选择器 |
| `P` | 跳转至产品（HS2）选择器 |
| `S` | 交换起运 ↔ 目的地 |
| `1` – `4` | 切换标签页（当前航线 / 替代方案 / 陆路 / 影响） |
| `↑` / `↓` | 在左侧栏的排名列表中导航 |
| `Enter` | 确认高亮选择 |
| `Cmd+,` | 复制可分享的 URL |
| `?` | 显示此帮助浮层 |

## 地图集成

当工作流打开时，底层地图会实时反映您的选择：

- 主航线被高亮显示。
- 当"替代方案"标签页活动时，绕行走廊以虚线叠加形式渲染。
- 视口会缩放以适应当前活动的航线集合。

工作流关闭时，地图会恢复其先前状态。

## 分层与门控

Route Explorer 为 **PRO** 功能。免费层访客可以打开工作流并填写输入栏，但在拉取数据时会遇到硬门控——左侧栏被模糊处理，标签页区域被替换为"Unlock route intelligence"卡片和 **Upgrade to PRO** 按钮，任何标签页中都不会渲染数值化数据。模态背后可见的底层地图会显示一条通用公开航线高亮，作为 PRO 功能预览。

门控路径由 `src/components/RouteExplorer/RouteExplorer.ts:342` 中的 `renderFreeGate()` 实现；若点击 Upgrade 按钮，会触发 `route-explorer:free-cta-click` 分析事件。点击 Upgrade 会进入 Pro 结账流程（或回退到 Pro 营销页面）。

## Route Explorer 背后的数据

Route Explorer 调用由 `SupplyChainService` 生成的 sebuf REST RPC。航线与影响调用均为 PRO 路径；免费用户可打开工作流，但拉取时的门控会用升级卡片替换数值结果。航线情报供应商路径由 `get-route-explorer-lane` 在服务端封装；浏览器代码应使用下方生成的 REST 路由。

| 工作流区域 | 方法 + 路由 | 支撑内容 |
|---|---|---|
| 当前 / 备选 / 陆路标签页 | `GET /api/supply-chain/v1/get-route-explorer-lane` | 主航线 id、航线几何、咽喉要道敞口、含几何的绕行选项、战争风险等级与阻断评分。 |
| 影响标签页 | `GET /api/supply-chain/v1/get-route-impact` | 航线价值、主要出口商份额、战略产品、韧性评分、依赖标识与 Comtrade 源状态。 |
| 公开咽喉要道上下文 | `GET /api/supply-chain/v1/get-chokepoint-status` | 当前咽喉要道状态、AIS 中断计数与地图其他位置显示的近期事件上下文。 |
| 咽喉要道历史下钻 | `GET /api/supply-chain/v1/get-chokepoint-history` | 针对展开的咽喉要道卡片的惰性过境计数历史。 |

有关 API 层契约，请参阅 [Supply Chain](/api/SupplyChainService.openapi.yaml) 和 [Shipping v2](/zh/api-shipping-v2) 参考页面。有关地图底层海上图层的更多信息，请参阅 [Maritime Intelligence](/zh/maritime-intelligence)。

## 相关工作流

- [Scenario Engine](/zh/scenario-engine)——如果 Route Explorer 回答的是"这条航线今天如何"，那么 Scenario Engine 回答的是"如果某个具体的阻断事件真的发生，会怎样"。
- [Maritime Intelligence](/zh/maritime-intelligence)——主地图上船舶、咽喉要道和走廊的图层级别视图。
